# 多线程事件驱动优化 - V1.1

**日期**: 2026-06-23  
**版本**: V1.0.2 → V1.1  
**重大升级**: NFC读卡改为多线程事件驱动

---

## 🎯 优化目标

**原问题**：NFC采用轮询方式，主循环必须等待读卡超时才能继续

**解决方案**：多线程事件驱动架构
- 后台线程持续轮询NFC
- 主线程等待事件队列（阻塞但不占CPU）
- 检测到卡片立即通知主线程

---

## 🏗️ 架构变化

### 优化前（轮询架构）❌
```
主线程:
  while True:
    检查环境() → 100ms
    读取NFC(timeout=1s) → 1000ms [阻塞，占用I2C]
    处理认证()
  
  总循环时间: ~1100ms/次
  CPU利用率: 中等
```

### 优化后（事件驱动架构）✅
```
后台线程:
  while True:
    读取NFC(timeout=0.5s)
    检测到卡 → 放入事件队列
    time.sleep(0.1s)  # 未检测到卡时休眠

主线程:
  while True:
    检查环境() → 100ms
    等待卡片事件(timeout=1s) → 阻塞但CPU空闲
    处理认证()
  
  CPU利用率: 极低
  响应延迟: <100ms
```

## 🔧 核心改动

### 1. 新增导入
```python
import threading  # 多线程
import queue      # 线程安全队列
```

### 2. 重写NFCReader类

**新增属性**：
```python
self.event_queue = queue.Queue(maxsize=10)  # 卡片事件队列
self.running = False                        # 线程运行标志
self.polling_thread = None                  # 后台线程对象
self.last_uid = None                        # 防重复读取
self.last_read_time = 0                     # 上次读卡时间
```

**新增方法**：
```python
start_polling()        # 启动后台线程
stop_polling()         # 停止后台线程
wait_for_card()        # 等待卡片事件（阻塞）
_polling_worker()      # 后台工作线程
```

### 3. 后台轮询逻辑

```python
def _polling_worker(self):
    """后台轮询工作线程"""
    while self.running:
        uid = self.nfc.read_passive_target_id(timeout=0.5)
        
        if uid:
            uid_str = format(uid)
            
            # 防止重复读取（2秒内同一张卡）
            if 需要上报:
                self.event_queue.put_nowait(uid_str)
                time.sleep(0.5)  # 检测到卡后短暂延迟
        else:
            time.sleep(0.1)  # 未检测到卡时休眠
```

**优势**：
- ✅ 短超时（0.5秒）+ 休眠，降低I2C占用
- ✅ 防重复逻辑，避免同一张卡多次触发
- ✅ 异常自动恢复

### 4. 主线程调用方式

```python
# 初始化时启动后台线程
self.nfc_reader.start_polling()

# 主循环中等待事件
uid = self.nfc_reader.wait_for_card(timeout=1.0)  # 阻塞，CPU空闲
if uid:
    handle_authentication(uid)

# 关闭时停止线程
self.nfc_reader.stop_polling()
```

## 🎯 架构优势

### 1. **真正的事件驱动**
- 主线程 `wait_for_card()` 使用 `queue.get()` 阻塞
- 阻塞期间CPU完全空闲（操作系统调度）
- 有卡片事件时立即唤醒，响应迅速

### 2. **防重复机制**
```python
# 同一张卡2秒内不重复上报
if uid_str != self.last_uid or (current_time - self.last_read_time) > 2.0:
    self.event_queue.put(uid_str)
```

### 3. **异常自动恢复**
```python
try:
    # 读卡操作
except Exception as e:
    logging.error(f"NFC轮询线程错误: {e}")
    time.sleep(1.0)  # 错误后延迟，避免错误循环
```

### 4. **优雅关闭**
```python
def stop_polling(self):
    self.running = False
    self.polling_thread.join(timeout=3.0)  # 等待线程退出
```
