# CPU优化 - V1.0.2

**日期**: 2026-06-23  
**版本**: V1.0.1 → V1.0.2

---

## 🎯 优化目标

**问题**：环境监测每次主循环都执行，占用过多CPU资源，影响NFC读卡响应速度

**解决方案**：改为定时触发，每5秒检测一次环境

---

## 🔧 优化内容

### 1. 新增配置常量
```python
# 环境监测配置
ENV_CHECK_INTERVAL = 5.0  # 环境监测间隔(秒) - 降低CPU占用
```

### 2. 添加时间戳记录
```python
class ToolCabinetController:
    def __init__(self):
        self.last_env_check = 0  # 上次环境检测时间戳
```

### 3. 修改待机状态处理逻辑
```python
def handle_idle_state(self):
    """处理待机状态"""
    # 定时检测环境（每ENV_CHECK_INTERVAL秒执行一次）
    current_time = time.time()
    if current_time - self.last_env_check >= ENV_CHECK_INTERVAL:
        env_data = self.env_controller.monitor_and_adjust()
        self.last_env_check = current_time
    
    # 等待用户认证（NFC读卡不受影响）
    uid = self.nfc_reader.read_card(timeout=NFC_READ_TIMEOUT)
    ...
```

## 🔍 工作原理

### 时间轴示意图

```
时间 →
0s    1s    2s    3s    4s    5s    6s    7s    8s    9s    10s
|-----|-----|-----|-----|-----|-----|-----|-----|-----|-----|
NFC   NFC   NFC   NFC   NFC   NFC   NFC   NFC   NFC   NFC    (每秒检测)
ENV                     ENV                     ENV           (每5秒检测)
```

### 主循环流程

```python
while running:
    # 1. 检查是否需要环境监测（每5秒）
    if time.time() - last_check >= 5.0:
        check_environment()  # I2C读取温湿度
        last_check = time.time()
    
    # 2. 立即执行NFC读卡（不等待）
    uid = nfc_read(timeout=1.0)  # 阻塞1秒或检测到卡
    
    # 3. 如果检测到卡，处理认证
    if uid:
        authenticate(uid)
```

## ⚙️ 可调参数

### ENV_CHECK_INTERVAL 建议值

| 场景 | 间隔 | 说明 |
|------|------|------|
| **生产环境** | 5-10秒 | 推荐，平衡响应速度和CPU占用 |
| **测试环境** | 3-5秒 | 快速验证环境控制逻辑 |
| **低功耗模式** | 30-60秒 | 极低功耗，适合电池供电 |
| **高精度监测** | 1-3秒 | 需要快速响应环境变化 |

### 当前配置
