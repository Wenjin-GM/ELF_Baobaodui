# STM32 PB0 单线槽位在位输出协议

更新时间：2026-07-03

用途：STM32 解码充电盒 S 线后，用 PB0 这一根 GPIO 向主控板输出四个槽位的“空/有电池”状态。

## 接线

```text
STM32 PB0 -> 主控板 GPIO 输入
STM32 GND -> 主控板 GND
```

PB0 是 3.3V CMOS 输出。主控板 GPIO 必须配置为输入，不能反向驱动 PB0。

## 输出内容

STM32 将充电盒原始状态字按槽位字段转换为 4-bit 在位掩码：

```text
bit0 = slot1_present
bit1 = slot2_present
bit2 = slot3_present
bit3 = slot4_present
```

判断规则：

```text
槽位字段 == 0 -> 空
槽位字段 != 0 -> 有电池
```

这里不限定“有电池”必须是某一个固定码型，因为充电盒后续可能用不同码型表示满电、两格电、一格电等状态。

当前 STM32 固件使用的原始字段掩码：

```text
slot1: raw bits 4..6
slot2: raw bits 7..9
slot3: raw bits 10..12
slot4: raw bits 13..14
```

为降低抖动，STM32 使用每槽独立积分滤波后更新 PB0 输出：

```text
检测到有电池：+2
检测到空：-1
分数 >= 4 -> 判有电池
分数降到 0 -> 判空
```

这可以过滤偶发的全空帧。

## PB0 脉宽协议

PB0 空闲时为高电平。

每帧格式：

```text
start: 低 300ms, 高 100ms
slot1: 短/长低脉冲, 高 100ms
slot2: 短/长低脉冲, 高 100ms
slot3: 短/长低脉冲, 高 100ms
slot4: 短/长低脉冲, 高 100ms
stop:  低 100ms, 高 700ms
```

数据位编码：

```text
空       -> 低 100ms
有电池   -> 低 300ms
```

主控板读取建议：

1. GPIO 配置为输入。
2. 等待 PB0 从高变低。
3. 测量每个低电平持续时间。
4. 第一个约 300ms 低脉冲是 start。
5. 后面 4 个低脉冲依次对应 slot1..slot4。
6. 低电平持续时间小于约 200ms 判空，大于约 200ms 判有电池。
7. 连续读取 2 到 3 帧一致后再更新业务状态。

轮询周期建议：

```text
10ms ~ 30ms
```

不需要高精度计时器；Linux 用户态 GPIO 轮询也能读。

## 当前验证记录

2026-07-03，当前充电盒状态为 2、3、4 号槽有电池，1 号槽空。STM32 USB CDC 调试输出验证结果：

```text
stable presence mask = 0xE
p1=0 p2=1 p3=1 p4=1
```

注意：同一在位状态下，原始 `word` 可能随充电格数等状态变化，例如出现 `0xA480`、`0xA700` 或偶发 `0x8000`。主控板不应依赖单个固定原始 word；应以 PB0 输出的稳定在位掩码为准。
