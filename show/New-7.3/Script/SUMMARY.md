# 智能安全工具柜触摸屏界面开发总结

## 项目概述

基于 **智能工具柜触摸屏界面框架大纲.md** 的要求，完成了 PyQt5 触摸屏界面的第一版实现。

**开发日期**: 2026-06-21  
**项目**: 宝宝队 ELF 2 / RK3588 智能安全工具柜  
**目标平台**: ELF2-Desktop 22.04 / RK3588

---

## 已实现的功能

### 1. 核心架构 ✅

- [x] **状态机驱动** (`state_machine.py`)
  - 9 种系统状态：STANDBY, AUTH_PENDING, USER_AUTHED, ADMIN_AUTHED, CABINET_OPEN, CHECKING_AFTER_CLOSE, ALARM_ACTIVE, MAINTENANCE, OFFLINE
  - 每个状态定义默认页面和允许访问的页面列表
  - 状态转换自动触发页面跳转
  - 支持用户权限管理（普通用户/管理员）

- [x] **主窗口框架** (`main_window.py`)
  - QStackedWidget 多页面管理
  - 顶部状态栏：系统名称、当前状态、登录用户、时间
  - 底部导航栏：根据状态动态生成，高亮当前页面
  - 支持点击导航和左右滑动切换页面
  - 滑动范围自动限制在当前状态允许的页面内

- [x] **模拟后端** (`mock_backend.py`)
  - 环境数据模拟（温湿度、风扇状态）
  - 工具状态模拟（五区检测结果）
  - 充电状态模拟（电池盒、4个电池位）
  - 认证事件模拟
  - 报警事件模拟
  - 定时更新机制

## 核心特性

### 状态机驱动的页面控制

```
STANDBY (待机)
  ↓ 认证成功
USER_AUTHED / ADMIN_AUTHED (已认证)
  ↓ 申请开柜
CABINET_OPEN (柜门打开)
  ↓ 关门
CHECKING_AFTER_CLOSE (盘点中)
  ↓ 盘点完成
STANDBY (返回待机) 或 ALARM_ACTIVE (报警)
```

每个状态自动控制：
1. 默认显示的页面
2. 底部导航栏显示的页面列表
3. 左右滑动允许访问的页面范围

### 双向交互支持

1. **点击导航** - 精确跳转
2. **滑动切换** - 快速浏览
3. **状态自动跳转** - 智能引导

### 模块化设计

```
main.py                    # 入口
  └─ MainWindow            # 主窗口
      ├─ StateMachine      # 状态机
      ├─ MockBackend       # 数据后端
      └─ Pages             # 8个页面
          ├─ DashboardPage
          ├─ AuthPage
          ├─ ToolsPage
          ├─ ChargingPage
          ├─ EnvironmentPage
          ├─ RecordsPage
          ├─ SettingsPage
          └─ DebugPage
```

## 运行测试

### 在 Windows 上测试

```bash
cd Show/Script
python main.py
```

### 在 Linux/ELF2 上测试

```bash
cd Show/Script
chmod +x run.sh
./run.sh
```

### 演示流程

1. 启动程序 → 总览页（未认证待机）
2. 点击"认证"导航 → 认证页
3. 点击"模拟管理员认证" → 自动返回总览页（管理员已认证）
4. 底部导航栏自动更新，显示所有可用页面
5. 观察环境数据每秒更新、工具状态每2秒更新、充电状态每3秒更新
6. 点击"申请开柜" → 自动跳转工具页（柜门打开状态）
7. 切换到设置页 → 调整阈值、进入维护模式
8. 在调试页 → 测试各模块、查看日志
9. 左右滑动测试页面切换
