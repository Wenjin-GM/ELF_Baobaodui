# Runtime Data Layout

This directory stores runtime outputs that may be reused by the UI, debugging scripts, and later analysis.

| Directory | Content |
| --- | --- |
| `inventory_images/` | Cabinet camera images, named by capture time, for example `YYYYMMDD_HHMMSS.jpg`. |
| `inventory_result/` | Inventory result JSON records, named by record time. |
| `ros_logs/` | Per-run ROS node stdout/stderr logs. Each launch creates `ros_logs/YYYYMMDD_HHMMSS/`. |
| `events/` | Cabinet event JSON records. |
| `auth/` | Authentication result JSON records. |
| `environment/` | SHT30/environment state JSON records. |
| `battery/` | Battery-box state JSON records. Currently filled by cabinet logic mock data until `battery_node` is implemented. |
| `actuator/` | Fan, lock, and buzzer actuator state JSON records. |
| `ui/` | Reserved for UI snapshots/state records. |
| `run/` | Runtime PID files and launch metadata. It is generated by launch scripts. |
